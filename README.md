GTX Sales Dashboard – CRM Analytics
A comprehensive Power BI dashboard designed to analyze sales performance, pipeline efficiency, and regional team productivity for the GTX product line. This project transforms raw CRM data into actionable insights for sales managers and stakeholders.

📊 Business Objectives
The goal of this dashboard is to provide a clear view of the sales lifecycle, from lead generation to closed deals. Key questions addressed include:

What is the total revenue generated across different product series (GTX, MG, GTK)?

Which sales agents and managers are exceeding their targets?

Which market sectors and office locations provide the highest value?

How does the sales pipeline look over time?

📁 Project Structure
The repository contains the following files:

/reports: Contains the crm dashbord 2.pbix file (Power BI Desktop file).

/data: Source CSV files used for the analysis:

sales_pipeline.csv: Transactional data including order dates and sales agents.

products.csv: Product details, series categories, and pricing.

accounts.csv: Client details, sectors, and geographic locations.

sales_teams.csv: Hierarchy of agents, managers, and regional offices.

README.md: Project documentation.

🛠️ Tools Used
Power BI Desktop: For ETL (Power Query), Data Modeling, and Visualization.

DAX (Data Analysis Expressions): Used for calculating custom measures like Total Revenue and Performance KPIs.

GitHub: For version control and project hosting.

🚀 Key Insights
Product Performance: Specialized analysis of the GTX series vs. MG and GTK series.

Regional Breakdown: Performance tracking across Central, East, and West regional offices.

Sector Analysis: Revenue distribution across sectors like Technology, Medical, Retail, and Finance.

📝 How to Use
Clone this repository to your local machine.

Open the reports/crm dashbord 2.pbix file using Power BI Desktop.

If the data links are broken, redirect the data source settings to the CSV files located in the /data folder.
